Requirements

The first step of utilizing DOORS is the documentation of user requirements. New requirements can be entered into DOORS using a word processor style interface. In a similar manner, system requirements can be manually entered as well.

IBM wanted to ease implementation of DOORS for tracking requirements at any stage in the project development. Previously captured requirements can also be imported into DOORS from many common office application file types:

  • Microsoft Word
  • Microsoft Excel
  • Microsoft Powerpoint
  • Plain text (.txt)
  • Rich Text Format (RTF)
  • FrameMaker

Once entered into DOORS, requirements can be tracked throughout the process using views, links, and traceability information.
